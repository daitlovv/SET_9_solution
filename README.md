# Эмпирический анализ алгоритмов сортировки строк

## Тестовые данные

Символы: A–Z, a–z, 0–9, `!@#%:;^&*()-. ` (75 символов);
Длина строк: 10–200 символов;
Размер массивов: 100–3000 с шагом 100;
Типы: случайный, обратно отсортированный, почти отсортированный, с общим префиксом.

## Выводы

**Время выполнения:**
Стандартный 'quicksort' медленнее всех во всех типах массивов, особенно на строках с общим префиксом;
'string_quicksort` и `msd_radix_switch` показали лучшее время во всех тестах;
Адаптированные алгоритмы растут медленнее стандартных с увеличением n.

**Посимвольные сравнения:**
`quicksort` делает на порядок больше сравнений, чем адаптированные алгоритмы;
`string_mergesort` и `msd_radix` минимизируют сравнения за счет LCP и bucket-распределения;
Наибольшая разница между стандартными и адаптированными алгоритмами видна на массивах с общим префиксом.

**Итог:**
Стандартные алгоритмы демонстрируют сложность выше O(n log n) по сравнениям, тк каждое сравнение строк стоит O(L); адаптированные алгоритмы не повторяют сравнение уже совпавших префиксов
