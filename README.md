# Эмпирический анализ алгоритмов сортировки строк

## Тестовые данные

- Символы: A–Z, a–z, 0–9, `!@#%:;^&*()-. ` (74 символа);
- Длина строк: 10–200 символов;
- Размер массивов: 100–3000 с шагом 100;
- Типы: случайный, обратно отсортированный, почти отсортированный, с общим префиксом.

## Измерения

- Время выполнения в микросекундах;
- Количество посимвольных сравнений;
- 3 прогона, среднее значение.
  
## Выводы

**Время выполнения:**
quicksort медленнее всех на всех типах. На prefix это виднее сильнее всего;
string_quicksort и msd_radix_switch самые быстрые на всех типах, растут медленнее остальных;
msd_radix без переключения заметно медленнее msd_radix_switch;
На prefix msd_radix догоняет mergesort и string_mergesort из-за накладных расходов от большого буфера бакетов при глубкой рекурсии.

**Посимвольные сравнения:**
quicksort делает на порядок больше сравнений чем все остальные, особенно на prefix, тк каждое сравнение строк повторно проходит общий префикс
mergesort также делает много сравнений на prefix по той же причине;
string_mergesort, string_quicksort, msd_radix_switch держатся близко друг к другу и значительно ниже стандартных;
msd_radix без переключения дает минимальные лексикографические сравнения, работает через бакет распределение без посимвольных сравнений напрямую. В варианте msd_radix_switch сравнения появляются при переключении на string_quicksort.

**Итог:**
Стандартные алгоритмы растут быстрее O(n log n) по сравнениям, тк каждое сравнение строк стоит O(L) => итоговая сложность O(nL log n); адаптированные алгоритмы не повторяют сравнение уже совпавших префиксов (сильнее всего заметно на массивах с общим префиксом).
